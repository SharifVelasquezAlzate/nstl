# nstl
### What is `nstl`?
`nstl` is a from-scratch implementation of C++'s STL. It is a work in progress (the STL Is huge, so I'll probably will only focus on implementing the data structures).

### Why `nstl`?
I have always liked to understand, and have always been intrigued by the inner workings of things. There are usually many things we give for granted, and the STL is one of them: if you want to use a string, just use `#include <string>` and things magically work! Same for vector, queue, and a long etcetera. This is completely fine, as one doesn't need to know how *everything* was written in order to write code or understand it, but I am just genuinely curious about this, and also consider that there are lots of advantages to caring about the "why?" of as many things as possible. The following are some of the reasons why I started this project:

1. **There is an inherent satisfaction to understanding and learning**: Being curious about something and discovering things on your own is an incredible experience, not only due to the inherent satisfaction of understanding, but because of all the learning that is involved in the process.

2. **Knowledge is power... and efficiency... and optimizations!**: Often times it is necessary to make decisions that will be hidden from the final user, and the STL is full of such decisions! Understanding the "why?" behind them is not only useful for learning (as mentioned previously) but a process that provides one with better decision making and improved awareness when using these concepts/data structures/classes/etc. to perform optimizations, identify situations where things should deviate from the usual approach, and many more things.

3. **Learn by doing**: In my opinion, there's no better way of learning than by throwing yourself into the void -- things may seem difficult or confusing at first, but it's only through the struggle of coming up with solutions on your own that you understand their reason of being in the first place.

4. **A never-ending story**: This is, almost by definition, a never-ending project, and thus, something I can always make progress on whenever I want to take a break from another project, or simply feel like coding in my free time.

### Rules
There are some rules I have decided to adopt for the development of this project. These are:

1. The STL should never be used (this includes C libraries like memcpy, strcpy, etc.), except for tests.
2. **No AI**: I have decided to not use AI anywhere in the code for this project. The only exception to this is setting up CMake files or metatools of a similar nature.
